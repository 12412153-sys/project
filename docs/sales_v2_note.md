# sales_v2 销售模块说明

## 1. 功能概述

`sales_v2` 是销售模式模块，负责普通顾客购买流程：翻页选择饮料、检查库存和状态、键盘输入金额、确认付款、找零、等待取货、交易完成或退款。

本版固定 4 种饮料：

| 编号 | 名称 | 价格 | 库存示例 | 选择页显示 |
|---:|---|---:|---:|---|
| 0 | COLA | 4 | 25 | `COLA4 25` |
| 1 | SODA | 5 | 12 | `SOdA5 12` |
| 2 | TEA  | 3 | 08 | `tEA 3 08` |
| 3 | H2O  | 2 | 30 | `H2O 2 30` |

价格只建议设置为 1~9，方便演示；库存支持两位数，建议 0~99。

---

## 2. 接口说明

### 2.1 基础输入

| 信号 | 位宽 | 说明 |
|---|---:|---|
| `clk` | 1 | 100 MHz 系统时钟 |
| `rst_n` | 1 | 低有效复位 |
| `sales_en` | 1 | 销售模式使能 |

### 2.2 按键输入

所有按键都默认已经在顶层消抖，进入本模块时是单周期脉冲。

| 信号 | 建议按键 | 作用 |
|---|---|---|
| `btn_confirm` | 确认键 | 确认选择、确认付款、确认取货 |
| `btn_prev` | 上一页 | 选择上一种饮料 |
| `btn_next` | 下一页 | 选择下一种饮料 |
| `btn_pay` | 保留 | 当前版本不用，金额用键盘输入 |
| `btn_cancel` | 取消键 | 取消订单或返回 |

### 2.3 键盘输入

| 信号 | 位宽 | 说明 |
|---|---:|---|
| `kbd_data` | [3:0] | PS/2 键盘解析出的 BCD 数字 0~9 |
| `kbd_valid` | 1 | 数字有效脉冲 |

付款状态下，每收到一个数字，就执行：

```verilog
paid_amount = paid_amount * 10 + kbd_data;
```

例如输入 `1` 再输入 `2`，金额变成 12。金额最大限制到 99。

### 2.4 商品数据输入

| 信号 | 位宽 | 说明 |
|---|---:|---|
| `price0~price3` | [7:0] | 四种饮料价格，建议 1~9 |
| `stock0~stock3` | [7:0] | 四种饮料库存，可显示两位数 |
| `enabled_mask` | [3:0] | 是否在售，1 表示在售，0 表示停售 |

### 2.5 给寄存器堆的输出

| 信号 | 位宽 | 说明 |
|---|---:|---|
| `drink_id` | [1:0] | 当前选中的饮料编号 |
| `sale_we` | 1 | 取货成功时拉高 1 个周期 |
| `sale_idx` | [1:0] | 成功售出的饮料编号 |
| `sale_amount` | [7:0] | 计入收入的金额，一般等于当前价格 |
| `refund_pulse` | 1 | 找零、取消或超时时拉高 1 个周期 |
| `refund_amount` | [7:0] | 退款或找零金额 |
| `exit_to_main` | 1 | 在选择页取消时请求返回主菜单 |

### 2.6 显示输出

| 信号 | 位宽 | 说明 |
|---|---:|---|
| `paid_amount` | [7:0] | 当前已输入金额 |
| `current_price` | [7:0] | 当前饮料价格 |
| `current_stock` | [7:0] | 当前饮料库存 |
| `led_out` | [15:0] | LED 输出 |
| `view_data` | [39:0] | 8 位数码管字符 ID，每位 5 bit |
| `state_code` | [3:0] | 当前状态 |
| `error_code` | [3:0] | 当前错误码 |

## 3. 状态机

| 状态 | 说明 |
|---|---|
| `ST_SELECT` | 选择饮料页面 |
| `ST_CHECK` | 检查是否在售、是否有库存、价格是否合法 |
| `ST_PAY` | 键盘输入金额并确认付款 |
| `ST_CHANGE` | 金额多于价格时显示找零 |
| `ST_WAIT` | 等待取货 |
| `ST_DONE` | 交易完成提示 |
| `ST_REFUND` | 退款提示 |
| `ST_ERROR` | 错误提示 |

主流程：

```text
SELECT -> CHECK -> PAY -> WAIT -> DONE
                  |      |
                  |      -> REFUND / timeout
                  -> CHANGE -> WAIT
                  -> ERROR
```

## 4. 数码管显示设计

### 4.1 选择页

格式：

```text
名字 + 1位价格 + 空格 + 2位库存
```

例子：

```text
COLA4 25
SOdA5 12
tEA 3 08
H2O 2 30
```

### 4.2 支付页

正常付款显示：

```text
PAY04 P4
```

含义：已经输入 04，价格为 4。

金额不足时显示：

```text
Add   03
```

含义：还需要补 3。

金额过多时进入找零页：

```text
rEt   02
```

含义：找零 2，然后进入等待取货。

### 4.3 取货、完成和错误

| 状态 | 显示 | 含义 |
|---|---|---|
| 等待取货 | `PUSH` | 提示顾客按确认键取货 |
| 完成 | `dOnE` | 交易完成 |
| 退款/找零 | `rEt xx` | 退款或找零金额 |
| 错误 | `Err   n` | 错误码 n |

---

## 5. 金额处理规则

付款时按确认键：

| 条件 | 处理 |
|---|---|
| `paid_amount < current_price` | 不出货，留在付款页，显示还需要补多少钱 |
| `paid_amount == current_price` | 进入等待取货 |
| `paid_amount > current_price` | 先找零，`refund_amount = paid_amount - current_price`，再进入等待取货 |

注意：只有在等待取货状态下按确认键，模块才输出 `sale_we=1`。也就是说，付款后如果顾客没有取货，不会扣库存，也不会计入收入。

## 6. LED 设计

| 状态 | LED |
|---|---|
| 选择页 | 亮当前饮料对应的 LED |
| 支付页 | 16 位付款进度条 |
| 找零页 | `16'h00ff` |
| 等待取货 | 流水灯，越接近超时速度越快 |
| 完成 | `16'haaaa` |
| 退款 | `16'h5555` |
| 错误 | `16'hf00f` |

## 7. 给 C 同学的集成提示

1. `view_data[39:0]` 仍然是 8 个 5-bit 字符 ID，需要 `seg7_mux` 转成七段码。
2. 顶层需要把键盘模块输出接入本模块：

```verilog
.kbd_data(kbd_data),
.kbd_valid(kbd_valid)
```

3. 库存信号需要支持两位数，所以 `stock0~stock3` 建议统一改为 `[7:0]`。
4. 价格仍然可以用 `[7:0]` 传入，但建议实际值限制在 1~9。
5. `btn_pay` 端口暂时保留，方便兼容原 top，但当前版本不使用它输入金额。
