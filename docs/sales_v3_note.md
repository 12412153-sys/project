# sales_v3 销售模块说明

## 1. 主要变化

`sales_v3` 是购物车版销售模式。和 v2 不同，v3 支持一次订单购买多瓶饮料，也支持不同饮料混合购买。

基本流程：

```text
选择饮料 -> 输入数量 -> 加入购物车 -> 继续选择或结算 -> 支付 -> 找零/等待取货 -> 扣库存并完成
```

如果数量超过库存，模块不会加入购物车，而是提示当前剩余库存，并让顾客重新输入数量。

---

## 2. 四种饮料和显示

价格仍建议设置为 1~9，库存支持两位数。

| 编号 | 名称 | 单价 | 库存示例 | 选择页显示 |
|---:|---|---:|---:|---|
| 0 | COLA | 4 | 25 | `COLA4 25` |
| 1 | SODA | 5 | 12 | `SOdA5 12` |
| 2 | TEA  | 3 | 08 | `tEA 3 08` |
| 3 | H2O  | 2 | 30 | `H2O 2 30` |

选择页格式：

```text
名字4位 + 1位单价 + 空格 + 2位剩余库存
```

这里的库存是 `stock - cart_qty`，也就是考虑购物车中已经选走的数量后的剩余库存。

---

## 3. 按键和键盘

| 输入 | 用途 |
|---|---|
| `btn_prev` | 上一种饮料 |
| `btn_next` | 下一种饮料 |
| `btn_confirm` | 确认选择、确认数量、确认付款、确认取货 |
| `btn_pay` | 在选择页进入结算付款 |
| `btn_cancel` | 取消当前步骤或退款返回 |
| `kbd_data/kbd_valid` | PS/2 键盘输入 BCD 数字 |

v3 中金额和数量都用键盘输入。板子按键只负责翻页、确认、结算和取消。

## 4. 接口重点

### 4.1 商品数据输入

```verilog
input wire [7:0] price0, price1, price2, price3;
input wire [7:0] stock0, stock1, stock2, stock3;
input wire [3:0] enabled_mask;
```

价格虽然是 8 bit 输入，但建议实际值限制在 1~9。库存是 8 bit，可以显示 0~99。

### 4.2 键盘输入

```verilog
input wire [3:0] kbd_data;
input wire       kbd_valid;
```

每次按下一个数字键，`kbd_valid` 拉高一个周期，`kbd_data` 给出 0~9。

### 4.3 寄存器堆写回

```verilog
output reg        sale_we;
output reg [1:0]  sale_idx;
output reg [7:0]  sale_amount;
```

v3 在取货确认后进入 `ST_COMMIT`，会根据购物车内容连续输出多次 `sale_we`。例如购物车里有 2 瓶 COLA 和 3 瓶 TEA，则会输出 5 个销售脉冲：

```text
COLA, COLA, TEA, TEA, TEA
```

这样 register_file 不需要一次处理多瓶，只要每收到一个 `sale_we` 就扣 1 瓶并加一次收入。

### 4.4 退款和找零

```verilog
output reg        refund_pulse;
output reg [11:0] refund_amount;
```

v3 的退款金额可能超过 255，所以这里改成 12 bit。找零、取消和超时都会使用这个接口。

## 5. 状态机

| 状态 | 说明 |
|---|---|
| `ST_SELECT` | 选择饮料，也可以按结算进入付款 |
| `ST_CHECK` | 检查当前饮料是否在售、是否有库存、价格是否合法 |
| `ST_QTY` | 输入当前饮料购买数量 |
| `ST_STOCK` | 数量超过库存时提示剩余库存 |
| `ST_FULL` | 总价超过 999 时提示订单过大 |
| `ST_ADDED` | 加入购物车后显示当前总价 |
| `ST_PAY` | 输入支付金额 |
| `ST_CHANGE` | 找零提示 |
| `ST_WAIT` | 等待顾客取货 |
| `ST_COMMIT` | 连续输出 sale_we，扣库存、加收入 |
| `ST_DONE` | 交易完成 |
| `ST_REFUND` | 退款提示 |
| `ST_ERROR` | 错误提示 |

## 6. 数码管显示

| 页面 | 显示例子 | 含义 |
|---|---|---|
| 选择页 | `COLA4 25` | COLA 单价 4，剩余库存 25 |
| 数量页 | `Cnt03 P4` | 买 3 瓶，单价 4 |
| 库存不足 | `StOC 25` | 当前最多只能买 25 瓶 |
| 总额过大 | `FULL 999` | 订单总价超过 999 |
| 加入成功 | `tot 012` | 当前购物车总价 12 |
| 支付页 | `0017t017` | 已付 17，总价 17 |
| 金额不足 | `Add 007` | 还需要补 7 |
| 找零 | `rEt 003` | 找零 3 |
| 等待取货 | `PUSH` | 按确认取货 |
| 完成 | `dOnE` | 交易完成 |
| 退款 | `rEt 017` | 退款 17 |
| 错误 | `Err   n` | 错误码 n |

金额统一按三位总价显示，最大支持 999。支付页左边显示 4 位已输入/已付金额，右边显示 3 位总价。

---

## 7. 金额处理规则

付款阶段按确认后：

| 条件 | 处理 |
|---|---|
| `paid < total` | 显示 `Add xxx`，提示还需要补多少钱，留在付款页 |
| `paid == total` | 进入等待取货 |
| `paid > total` | 输出 `refund_pulse`，找零 `paid-total`，然后进入等待取货 |

如果顾客在付款阶段长时间无操作，会退还已经输入的钱并返回销售主页。

---

## 8. 超时处理

v3 的人性化超时规则：

| 当前阶段 | 超时行为 |
|---|---|
| 数量输入 | 清空当前输入和购物车，回到选择页 |
| 支付阶段 | 如果已经输入金额，则退款；然后回到选择页 |
| 等待取货 | 退款订单总额，不扣库存、不计收入 |
| 提示页 | 自动回到选择页 |

只有进入 `ST_COMMIT` 并输出 `sale_we` 后，才真正扣库存、计入收入。
