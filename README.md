# 基于 FPGA 的饮料售货机 —— 系统框架设计文档

## 1. 项目总览

本系统是一个基于 FPGA 的饮料售货机，支持**销售**与**管理**两种工作模式，通过 EGO1 板载外设完成人机交互。

目前支持
- 4种饮料: COLA、SODA、TEA、H2O（缩写）
- 库存最大9（寄存器文件 [3:0]，顶层零扩展为 [7:0] 传给销售模块）
- 整数价格 1~9
- 上电后开始累计营收额（初始化为0），每笔交易按瓶逐一计入
- 密码8 bit BCD（键盘输入两位数字，多按几位以最后两位为准）
- **购物车模式（sales_v3）**：一次订单可购买多种/多瓶饮料，按 S3 结算
- 销售和管理模式金额/数量/密码均通过 PS/2 键盘输入
- 声音大小已实现占空比50%（物理硬件最大），只能修改频率

数码管显示格式示例
- sale 选择页: `COLA4 05`（COLA 价格4，库存5）
- sale 数量页: `Cnt03 P4`（买3瓶，价格4）
- sale 支付页: `0012t012`（已付12，总价12）
- admin 密码页: `----C0dE`
- admin 库存页: `C0dA__05`（COLA 库存5）

### 1.1 全局状态机

顶层 FSM 有三个状态，通过 `sys_state[1:0]` 寄存器维护：

| 状态 | 编码 | 描述 |
|------|------|------|
| `MAIN_MENU` | `2'd0` | 数码管显示 "r1-0S-1A"；SW[0]=0 选销售，SW[0]=1 选管理；S2 确认进入 |
| `SALE_MODE` | `2'd1` | 使能 `sales_mode`，数码管/LED 由销售模块驱动 |
| `ADMIN_MODE` | `2'd2` | 使能 `admin_mode`，数码管由管理模块驱动 |

状态转换：
```
MAIN_MENU --(S2, SW[0]=0)--> SALE_MODE  --(S6 soft exit)--> MAIN_MENU
MAIN_MENU --(S2, SW[0]=1)--> ADMIN_MODE --(S6 soft exit)--> MAIN_MENU
```

- **S6（btn[5], P15，低有效）软退出**：在 SALE_MODE 或 ADMIN_MODE 任意时刻按 S6，`sys_state` 立即返回 `MAIN_MENU`，`register_file` 数据（价格/库存/营收/密码）**保持不变**。  
  `drink_vending_top` 内部对 `btn[5]`（P15，低有效）取反后送入消抖模块，`btn_d[5]` 仍为高有效单周期脉冲。  
- **S5（PROG_B，专用配置引脚）全局复位**：按下后 FPGA 重新加载比特流，等效于完全上电，所有寄存器恢复出厂默认值。`rst_n` 已在顶层绑定为常数 `1'b1`（无外部引脚），`register_file` 通过 Verilog `initial` 块设定上电初始值。  
- `sales_v3.exit_to_main` 在销售主页（ST_SELECT）按 S0(cancel) 时清空购物车并保持在销售模式。  
- `admin_mode` 在 S_VIEW 首项按 S3 保持不变（无操作）；S_ALARM 按 S2 解除报警后回 S_IDLE → S_AUTH（继续留在管理模式）。

## 2. EGO1 硬件资源分配

### 2.1 输入

| 物理外设 | 顶层端口 | 说明 |
|----------|----------|------|
| 100 MHz 时钟 (P17) | `clk` | 系统时钟 |
| 配置擦除按键 S5 (PROG_B) | —（无用户端口） | **全局复位**：重新加载比特流 = 完全上电复位 |
| 按键 S2 (R15) | `btn[2]` | 确认/保存/报警解除 |
| 按键 S6 (P15，低有效) | `btn[5]` | **软退出**：从销售/管理模式返回主菜单（数据保留）；顶层对 `btn[5]` 取反后消抖 |
| 按键 S0 (R11) | `btn[0]` | 属性向后翻页（admin）/ **清空购物车**（sale SELECT）/ 取消付款→退款（sale PAY/WAIT） |
| 按键 S3 (V1)  | `btn[3]` | 属性向前翻页（admin）/ **结算付款**（sale） |
| 按键 S4 (U4)  | `btn[4]` | 饮料ID+（admin）/ 上一个饮料（sale） |
| 按键 S1 (R17) | `btn[1]` | 饮料ID-（admin）/ 下一个饮料（sale） |
| PS2_CLK (K5) | `ps2_clk` | PS/2 键盘：密码输入（admin）+ **金额/数量输入**（sale） |
| PS2_DATA (L4) | `ps2_data` | PS/2 键盘数据线 |
| 拨码开关 SW0~SW7 (R1~P5) | `sw[7:0]` | 见下表 |

**拨码开关用途**

| 开关位 | 主菜单 | 销售模式 | 管理模式 |
|--------|--------|----------|---------|
| SW[0] (R1) | 模式选（0=销售, 1=管理） | — | — |
| SW[1] (N4) | — | — | 修改开关（1=进入编辑子态） |
| SW[2..7] | — | 未使用 | — |

> 销售模式金额和数量均通过 PS/2 键盘输入（数字键 0–9），不再使用拨码开关。

### 2.2 输出

| 物理外设 | 顶层端口 | 说明 |
|----------|----------|------|
| 数码管 DN0 段线 (B4,A4,A3,B1,A1,B3,B2,D5) | `seg0[7:0]` | 右侧4位 {dp,g,f,e,d,c,b,a}，**高有效** |
| 数码管 DN1 段线 (D4,E3,D3,F4,F3,E2,D2,H2) | `seg1[7:0]` | 左侧4位 {dp,g,f,e,d,c,b,a}，**高有效** |
| 位选 BIT1~BIT8 (G2,C2,C1,H1,G1,F1,E1,G6) | `an[7:0]` | 8位位选，**高有效**，an[0]=最右位 |
| 音频 PWM (T1) | `aud_pwm` | 蜂鸣器方波输出 |
| 音频 SD# (M6) | `aud_sd` | 音频使能，常输出 1 |
| LED D1 组 (K3,M1,L1,K6,J5,H5,H6,K1) | `led[7:0]` | 进度条/流水灯/状态 |
| LED D2 组 (K2,J2,J3,H4,J4,G3,G4,F6) | `led[15:8]` | 进度条/流水灯/状态 |

**（VGA 接口引脚已在 XDC 中注释备用，启用 Bonus 时取消注释即可）**

## 3. 模块层级总览

```
drink_vending_top
├── btn_debounce   × 6   (消抖，每个按键一个实例，S0–S4 + S6/P15)
├── ps2_keyboard         (PS/2 → BCD digit + valid，admin 和 sale 共享)
├── register_file        (中央存储，位于 project_sub/)
├── admin_mode           (管理模式 FSM，位于 project_admin/)
├── sales_v3             (购物车销售模式 FSM，位于 project_sale/)
├── seg7_mux             (8 位数码管时分复用驱动)
└── buzzer_driver        (蜂鸣器报警音生成)
```

## 4. 子模块详细说明

### 4.1 `btn_debounce.v`

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `CNT_MAX` | 2,000,000 | 消抖时间 20ms（100MHz） |
| `CNT_WIDTH` | 25 | 计数器位宽 |

输出 `btn_out` 为**高有效单周期脉冲**：仅在按键持续按下 20ms 时输出一次高电平。已消除抖动，顶层直接使用。  
S0–S4 为高有效按键，直接接入；S6（P15）为低有效，顶层实例化时输入取反（`.btn_in(~btn[5])`），输出 `btn_d[5]` 仍为高有效。

### 4.2 `ps2_keyboard.v`

- 接收 PS/2 11-bit 帧（start + 8 data + parity + stop），数据 LSB 先至。
- 支持 break 码（0xF0 前缀）过滤：仅 make 码触发输出。
- 输出 `kbd_data[3:0]`（BCD 0-9）和 `kbd_valid`（1 周期脉冲）。
- 数字键扫描码（Set 2）：0=0x45，1=0x16，2=0x1E，3=0x26，4=0x25，5=0x2E，6=0x36，7=0x3D，8=0x3E，9=0x46。

### 4.3 `seg7_mux.v`

- 输入：`disp_data[39:0]` = 8 × 5-bit 字符 ID，打包格式 `{id7, id6, ..., id0}`（id7=最左位）。
- 字符 ID 与 `sales_mode.v` 中 `CH_*` localparams 一致（0-9=数字，10=A，11=b，...，23=blank，24=-）。
- 扫描频率：100MHz / (100MHz/800Hz) = 100Hz/digit，视觉稳定无闪烁。
- 输出 `an[7:0]`（高有效位选）、`seg0/seg1[7:0]`（高有效段码 `{dp,g,f,e,d,c,b,a}`）。
- 位选 0-3 对应 DN0（右组，驱动 seg0），位选 4-7 对应 DN1（左组，驱动 seg1）。

**Admin view_data 转换**（在顶层进行）：  
`admin_mode` 输出 32-bit（8 nibble）display，顶层用 `nib2id()` 函数将每个 nibble 转换为 5-bit 字符 ID：nibble 0x0-0xE → char ID 0-14，nibble 0xF → char ID 23（blank）。

### 4.4 `buzzer_driver.v`

- `alarm_trigger=1` 时生成双音交替报警音（880Hz / 440Hz，250ms 切换）。
- `aud_sd` 常高（使能板上音频放大器）。
- `alarm_trigger=0` 时 PWM 输出为低（静音）。

## 5. 顶层按键路由汇总

| 物理按键 | 端口 | MAIN_MENU | SALE_MODE | ADMIN_MODE |
|---------|------|-----------|-----------|------------|
| S2 (R15) | `btn[2]` | 确认进入模式 | 确认选择/确认付款/确认取货 | confirm/save/解除报警 |
| S6 (P15，低有效) | `btn[5]` | — | **软退出→MAIN_MENU（数据保留）** | **软退出→MAIN_MENU（数据保留）** |
| S0 (R11) | `btn[0]` | — | 清空购物车(SELECT) / 取消付款→退款(PAY/WAIT) | 属性下翻(next_attr) |
| S1 (R17) | `btn[1]` | — | 下一个饮料 | 饮料ID减(id_dec) |
| S3 (V1)  | `btn[3]` | — | **结算/进入付款** | 属性上翻(prev_attr) |
| S4 (U4)  | `btn[4]` | — | 上一个饮料 | 饮料ID加(id_inc) |
| S5 (PROG_B) | —（专用配置引脚） | 全局复位（重新加载比特流，所有数据归零） | ← | ← |
| PS/2 键盘 | kbd | — | **数量/金额输入（0–9键）** | 密码输入（0–9键） |

## 6. 数码管显示格式汇总

| 模式 | 典型显示 | 说明 |
|------|---------|------|
| 主菜单 | `r1-0S-1A` | SW[0]=0→销售(S)，SW[0]=1→管理(A) |
| 销售-选择 | `COLA4 05` | 饮料名+单价+空格+库存(2位) |
| 销售-检查 | ` CHEC   ` | 自动跳转，短暂显示 |
| 销售-数量 | `Cnt03 P4` | 已输入数量3，单价4 |
| 销售-库存不足 | `StOC  05` | 剩余最多可买5瓶 |
| 销售-订单过大 | `FULL 999` | 总价超过999 |
| 销售-已加购物车 | `tot 012` | 当前购物车总价12 |
| 销售-支付 | `0012t012` | 已付12，总价12 |
| 销售-金额不足 | `Add  007` | 还需补7 |
| 销售-找零 | `rEt  003` | 找零3 |
| 销售-等待取货 | `PUSH    ` | 按S2确认取走 |
| 销售-出货中 | ` OUt    ` | 正在扣库存/计收入 |
| 销售-完成 | `dOnE    ` | 交易完成 |
| 销售-退款 | `rEt  017` | 退款17 |
| 销售-错误 | `Err     n` | 错误码 n |
| 管理-密码 | `----C0dE` | 等待键盘输入密码 |
| 管理-库存 | `C0dA__05` | COLA，库存5 |
| 管理-价格 | `C0dA__04` | COLA，价格4 |
| 管理-状态 | `C0dA___1` | COLA，在售(1)/停售(0) |
| 管理-营收 | `0014____` | 累计实收20元(hex 0x14) |
| 管理-报警 | `AAAAAAAA` | 密码连续输错3次 |
