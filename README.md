## 4. 系统介绍

### 4.1 系统整体架构框图

本系统是一个基于 FPGA 的饮料售货机，支持 **销售** 与 **管理** 两种工作模式，通过 EGO1 板载外设完成人机交互。支持 4 种饮料（COLA、SODA、TEA、H2O），单价 1~9，库存 0~99。

### 4.2 顶层模块与子模块关系

```
drink_vending_top                       （顶层：三态 FSM + 输入路由 + 显示/LED 复用）
├── btn_debounce        × 6             （按键消抖，每键一实例；S6/P15 低有效取反后接入）
├── ps2_keyboard        × 1             （PS/2 11-bit 帧解码 → BCD 数字 + valid 脉冲；销售/管理共享）
├── register_file       × 1             （中央存储；admin 与 sale 各一写口，并发读出）  [project_sub]
├── admin_mode          × 1             （管理模式 FSM；密码鉴权/查看/修改/报警）        [project_admin]
├── sales_v3            × 1             （购物车销售 FSM；多瓶多种购买/支付/找零/超时）  [project_sale]
├── seg7_mux            × 1             （8 位数码管时分复用驱动）
├── clk_wiz_0           × 1             （修改Clocking Wizard IP后生成，输出作为VGA时钟的输入）
├── buzzer_driver       × 1             （报警音生成：1320/1760/2200 Hz 三音递升循环）
└── vga_display         × 1             （640×480 字符网格富文本界面）         [project_top/project_vga]
```

顶层 `sys_state` 决定数码管/LED/VGA 显示哪个模块的数据；`register_file` 作为唯一数据源被销售、管理、显示三方共享，保证“加入购物车 / 出货扣库存 / 管理修改”等操作在数码管与 VGA 上**实时一致**。

**注意：clk_wiz_0.v并不是我们自己书写的，是VGA需要25.175MHz的时钟频率，在修改Clocking Wizard IP之后生成一个`clk_wiz_0`模块，在顶层模块也会被实例化，输出的时钟信号作为VGA的时钟输入。**

### 4.3 各子模块功能及典型接口

#### 4.3.1 register_file（中央寄存器堆）
- **功能**：保存 4 种饮料的价格、库存、在售掩码、累计营收、管理密码；提供并发读，管理与销售各一写口（写优先级 admin > sale）。
- **出厂默认值**：COLA ¥4/库存 5，SODA ¥5/库存 6，TEA ¥3/库存 8，H2O ¥2/库存 9；`enabled_mask=4'b1111`，营收 0，密码 `8'h61`（键盘“6”“1”）。
- **典型接口**：
  ```verilog
  // 管理写口
  input  admin_we;  input [1:0] admin_upd_type;  // 01=改价 10=改库存(≤99) 11=切换在售
  input  [7:0] admin_upd_data; input [1:0] admin_drink_id;
  // 销售写口（每瓶一次脉冲）
  input  sale_we;   input [1:0] sale_idx;  input [7:0] sale_amount;
  // 并发读
  output [7:0] price0..3; output [7:0] stock0..3; output [3:0] enabled_mask;
  output [15:0] total_revenue; output [7:0] password;
  ```

#### 4.3.2 sales_v3（购物车销售模式）
- **功能**：一次订单支持多种/多瓶饮料；选择→输入数量→加入购物车→结算→支付→找零/等待取货→出货扣库存计收入。库存不足、订单过大、停售、价格非法、超时、无效编号均有提示。
- **状态机**：`ST_SELECT / ST_CHECK / ST_QTY / ST_STOCK / ST_FULL / ST_ADDED / ST_PAY / ST_CHANGE / ST_WAIT / ST_COMMIT / ST_DONE / ST_REFUND / ST_ERROR`。
- **错误码**：1=停售，2=库存不足，3=金额不足，4=价格非法，5=超时，6=无效编号（选择页键盘输入 0 或 5~9）。
- **出货机制**：`ST_COMMIT` 按购物车内容连续输出多个 `sale_we` 脉冲（例如 2 瓶 COLA + 3 瓶 TEA → 5 个脉冲），寄存器堆每收到 1 个脉冲扣 1 瓶并计 1 次收入。
- **典型接口**：`btn_confirm/prev/next/pay/cancel`、`kbd_data/kbd_valid`、价格/库存/使能输入；输出 `sale_we/idx/amount`、`refund_pulse/amount`、`drink_id`、`cart0..3`、`paid_amount/total_amount`、`view_data[39:0]`、`state_code`、`error_code`、`led_out`。

#### 4.3.3 admin_mode（管理模式）
- **功能**：PS/2 密码鉴权（默认 0x61）；通过查看子态浏览各饮料库存/价格/在售状态/累计营收；SW[1] 上拨进入修改子态用键盘改值；密码输错时在数码管显示 `FAIL    `（5 秒保持，由参数 `FAIL_HOLD=500_000_000` 控制），第 3 次输错后进入持续报警（有蜂鸣器）。
- **状态机**：`S_IDLE / S_AUTH / S_FAIL / S_VIEW / S_MODIFY / S_SAVE / S_ALARM`。
  - `S_FAIL`：单次密码错误后暂停态；显示 `FAIL    `，无蜂鸣器，持续 FAIL_HOLD 个时钟周期后自动返回 `S_AUTH`。
  - `S_ALARM`：累计第 3 次错误后触发；持续报警直到 S2 解除。
- **典型接口**：`admin_en`、`sw_modify`、`kbd_data/valid`、`btn_confirm/next_attr/prev_attr/id_inc/id_dec`、`current_stock/price`、`total_revenue`、`password_in`；输出 `view_data[39:0]`、`write_en`、`update_type_out`、`update_data`、`drink_id`、`alarm_trigger`、`attr_sel_out`、`show_total_out`、`state_out`。

#### 4.3.4 外设/显示子模块
- **btn_debounce**：20 ms 消抖（`CNT_MAX=2,000,000`@100MHz），输出高有效单周期脉冲。
- **ps2_keyboard**：解析 PS/2 11-bit 帧，过滤 break 码（0xF0），输出 `kbd_data[3:0]`（0–9）和 `kbd_valid` 脉冲。
- **seg7_mux**：输入 `disp_data[39:0]`=8×5-bit 字符 ID，时分复用扫描 8 位数码管；`an[0]`=最右位。
- **buzzer_driver**：`alarm_trigger=1` 时生成 1320/1760/2200 Hz 三音递升循环报警音（每音约 150 ms 轮换），`aud_sd` 常高。
- **vga_display**：640×480@60Hz，像素时钟 25.175 MHz（由板载 MMCM Clocking Wizard IP 精确生成，确保显示器锁相稳定），40×30 字符网格、16×16 像素单元（8×8 ASCII 字模 2 倍放大）；按 `sys_state` 渲染主菜单/销售/管理三套界面，数据全部来自寄存器堆/销售/管理总线，**实时刷新**。接收两路时钟：`clk`（100 MHz，用于数据预计算）和 `pclk`（25.175 MHz，用于像素输出）。

```

   ▲  │pay&total>0                  │                         │                 ├─是─▶ ST_ADDED ─▶（继续/结算）
   │  └────────────────▶ ST_PAY     ├─停售/空/价格非法─▶ ST_ERROR              └─否─▶ ST_STOCK ─▶（重输）

```


## 5. 使用详细说明

本节给出 ≥2 个复杂测试场景的完整复现步骤，以及全部 Bonus 功能的测试方法。出厂默认值：COLA ¥4/5，SODA ¥5/6，TEA ¥3/8，H2O ¥2/9，密码 `61`。

> 注意：每次上电（按 S5/PROG_B 重新加载比特流）后寄存器恢复出厂值；S6 软退出返回主菜单但保留数据。

### 5.1 复杂测试场景一：多种多瓶购物车 + 超额付款找零 + 出货扣库存

| 步骤 | 操作 | 预期现象（数码管 / VGA / LED） |
|------|------|-------------------------------|
| 1 | 上电，SW[0]=0，按 S2 | 进入销售，选择页 `COLA4 05`；VGA 显示饮料列表，COLA 高亮 |
| 2 | PS/2 键盘按 `1`（或 S1 翻页选 COLA），按 S2 确认 | 进入数量页 `Cnt00 P4` |
| 3 | 键盘输入 `2`，按 S2 加入购物车 | `tot 008`（2×¥4=8）；VGA 购物车显示 COLA×2，左侧 COLA 库存仍为 5（出货前不变） |
| 4 | 键盘按 `3` 跳到 TEA，按 S2 确认，输入 `3`，按 S2 | 加入 3 瓶 TEA；`tot 017`（8+3×3=17）；VGA 购物车显示 COLA×2、TEA×3 |
| 5 | 按 S3 结算 | 进入支付页 `0000t017`（已付 0，总价 17） |
| 6 | 键盘输入 `2`、`0`（投 20），按 S2 | 20>17，找零页 `rEt 003`，随后进入等待取货 `PUSH   5`（倒计时 5s） |
| 7 | 在倒计时内按 S2 取货 | `OUt`→`dOnE`；出货阶段连续 5 个 `sale_we`：COLA 库存 5→3，TEA 库存 8→5；营收 +17 |
| 8 | 软退出 S6 → SW[0]=1 → S2 进管理，密码 `61`，浏览到营收页（键盘按 `5` 或 S0 翻到末尾）| 数码管显示 `t  00017`（`t` 为标识符，右 5 位为十进制营收）；COLA 库存 3、TEA 库存 5；VGA 管理界面 TOTAL REVENUE $00017 |



### 5.2 复杂测试场景二：库存不足 / 停售 / 无效编号 / 取货超时退款


| 步骤 | 操作 | 预期现象 |
|------|------|----------|
| 1 | 进入销售，键盘 `4` 选 H2O（库存 9），S2 确认，输入数量 `9` 以上（如直接尝试更多） | 数量超库存 → `StOC  09`（最多可买 9），要求重输 |
| 2 | 管理模式将 TEA 停售（见 5.3 修改流程后退出再进销售） | 选择页 TEA 显示 `tEA 3 --`（库存位点亮中间横段表示停售）；VGA 横幅可触发 `SOLD OUT` |
| 3 | 选择页用键盘输入 `0` 或 `5`~`9`（越界编号） | 报错 `Err   6`；VGA 黄底红字横幅 `INVALID CODE`；提示页超时/按键后回选择页 |
| 4 | 正常加入若干饮料并结算，进入等待取货 `PUSH   5` 后**不按任何键** | 倒计时 5→0，超时进入 `rEt XXX` 退款订单总额；**库存不变、营收不变**（仅 ST_COMMIT 才真正扣减） |
| 5 | 支付页投入部分金额后长时间不操作 | 自动退款已投金额并返回选择页 |


### 5.3 管理模式复现：鉴权 / 修改 / 报警

| 步骤 | 操作 | 预期现象 |
|------|------|----------|
| 1 | 主菜单 SW[0]=1，按 S2 | 进入管理，密码页 `COdE  00`（左 4 位固定显示 `COdE`，右 2 位显示已输入密码，初始为 `00`） |
| 2 | 键盘输入 `6` `1`，按 S2 | 鉴权通过，进入查看页，默认显示 COLA 库存：`1COLA 05`（最左位为饮料编号 1，其后为名称和两位库存） |
| 3 | S1 饮料 ID +1，S4 饮料 ID −1；S0/S3 翻属性（库存/价格/状态/营收）；键盘 `1`~`4` 直跳饮料、`5` 跳营收 | 数码管与 VGA 同步显示对应数据（如 COLA 单价页显示 `1COLA P4`，状态页 `1COLA  1`） |
| 4 | SW[1] 上拨进入修改，键盘输入新值（十进制累加，库存最大 99），按 S2 保存 | 数据写回寄存器堆，销售页立即生效 |
| 5 | 退出重进，依次输错密码 3 次 | 第 1、2 次错误：数码管显示 `FAIL    `，约 5 秒后自动回密码输入界面（无蜂鸣器）；第 3 次错误：显示 `FAIL    `，LED `0xAAAA`，蜂鸣器 1320/1760/2200 Hz 三音递升持续响；按 S2 解除后蜂鸣停止，返回密码页 |

## 6. 开源、AI 使用说明与思考总结

### 6.1 开源使用说明

- **EGO1 引脚约束**：参考板卡官方的引脚分配，自行编写 `vending.xdc`（全部 `LVCMOS33`）。
- **VGA 时序**：采用标准 640×480@60Hz VGA 时序参数（前后沿/同步宽度为公开标准），像素时钟 25.175 MHz 由板载 MMCM（Clocking Wizard IP）从 100 MHz 精确生成，保证各类显示器均能锁相。
- **PS/2 协议**：依据公开的 PS/2 键盘 11-bit 帧格式与 Set 2 扫描码表实现解码。

### 6.2 AI 使用说明
除法用减法替代，多个时钟流水线，键盘输入缓冲
- 使用 AI 辅助工具协助（ChatGPT, Claude, Gemini）：代码审查与梳理、文档起草与排版，以及一些具体模块的起草，开发者后续基于此进行学习和修改。具体辅助部分包括：
  1. **PS/2 键盘模块**：一开始我们如果只用拨码开关输入，操作会比较麻烦，所以项目中加入了 PS/2 键盘。PS/2 键盘涉及扫描码、11 位数据帧、按键按下和释放等细节，直接写起来比较容易出错。AI 帮助我们梳理了键盘输入的大致流程，包括如何接收扫描码、如何过滤按键释放码 `8'hF0`，以及如何把数字键映射成 0–9 的 BCD 数据。之后我们把这个模块接入销售模式和管理模式，用来输入数量、金额和密码。
  2. **VGA 显示模块**：VGA 模块是整个项目里代码量比较大的部分之一。它不仅要产生行同步和场同步信号，还要根据当前系统状态显示不同界面，比如主菜单、商品列表、购物车、管理信息和错误提示等。AI 在这部分主要帮助我们搭建了 VGA 显示的基本框架，包括像素时钟、行列计数、字符显示和颜色控制。同时在调试过程中AI帮找出了输出时钟和VGA所需时钟频率不同导致无法显示的问题并告知解决办法（修改Clocking Wizard IP）。
  3. **尝试消除 Error和Warning**：上板综合时出现两类 `Timing 38-282` 时序违例，AI 协助定位并修复：
     - **跨时钟域误报**：100 MHz 系统时钟与 MMCM 生成的 25.175 MHz 像素时钟之间没有公共周期（比值 3.97），Vivado 会对所有跨域信号计算 ~0.035 ns 的不可能满足的建立时间，实际上这些信号是缓慢变化的控制/数据总线，跨域无害。在 `vending.xdc` 中加入 `set_clock_groups -asynchronous` 约束，声明两个时钟域异步，彻底消除误报，域内路径仍正常计时。
     - **16-bit 营收除法路径过长**：`admin_mode.v` 中对 16-bit 营收做 5 位十进制展开（`/10000`、`/1000` …）是长组合路径，AI 将其改为两级流水线比较链（Stage 1 提取高两位和余数，Stage 2 提取低三位），每级均在 10 ns 内完成，彻底消除了这条路径的时序违例。
- 所有 AI 生成或建议的内容均经过小组成员人工审阅（学习了部分课程没有涉及到的语法和思路）、在 EGO1 上验证后才采纳。

### 6.3 Bonus 设计说明

1. **VGA 富文本显示界面（Bonus）**：640×480@60Hz，40×30 字符网格，自绘 8×8 ASCII 字模 2 倍放大；A_NORM/A_TITLE/A_HEAD/A_SEL/A_GUIDE/A_ERR 六种属性配色。有三套显示界面，包括主菜单、销售（饮料列表+库存+购物车+总额+操作指引+错误横幅）、管理（属性表+营收）。还有一些辅助操作的设计，如：选中饮料高亮、购物车数量、支付/找零、错误横幅均随操作即时刷新。
1. **PS/2 键盘输入（Bonus）**：解码 PS/2 帧得到 0–9 数字，用于金额/数量/密码输入及选择页 1~4 快速选饮料。
2. **蜂鸣器报警（Bonus）**：密码每次输错显示 `FAIL    ` 约 5 秒提示；连续 3 次错误后触发持续报警（1320/1760/2200 Hz 三音递升循环），按 S2 解除。
3. **购物车多瓶/多种购买**：一次订单混合购买多种多瓶，等待取货 5s 倒计时，超时退款不落账。
4. **密码**：进入管理模式之前需要管理者输入 8 bits密码。


### 6.4 思考总结
1. **整体结构设计**：我们把价格、库存、在售状态、密码和营收这些数据统一放在 `register_file` 里面。这样销售模块、管理模块和显示模块都从同一个地方读数据，可以避免出现“数码管/VGA 显示的库存”和“实际扣减的库存”不一致的问题。显示模块只负责读取和显示，不直接修改数据，这样整体逻辑也更清楚。

2. **状态机设计**：项目里销售和管理流程都比较长，所以除了把销售和管理模式在顶层中当作两个状态，我们还把这两个模式下的功能实现拆成多个小状态。比如销售模块中有选择饮料、输入数量、加入购物车、付款、找零、等待取货、出货和退款等状态；管理模块中有密码验证、查看、修改和报警等状态。这样虽然状态数量变多了，但每一步做什么会更清楚，后面调试也方便一些。

3. **遇到的问题与解决**：项目中遇到的问题主要集中在外设和模块联调上。比如 PS/2 键盘需要过滤按键释放码 `8'hF0`，否则一次按键可能会被识别多次；数码管一开始也出现过左右位顺序和显示内容对不上的情况，需要重新确认位选顺序；VGA 显示部分因为像素时钟、行列计数和字符显示逻辑比较多，调试起来也比较复杂。这些问题最后都是通过查阅资料、AI 辅助分析、修改代码和反复上板测试逐步解决的。

4. **可改进方向**：目前项目已经能完成基本售货机流程和部分增强功能，但还有一些可以继续完善的地方。例如现在价格和库存显示范围比较有限，后续可以支持三位数库存，支持价格是小数。此外还可以增加更多饮料种类，或者进一步优化 VGA 显示界面和时序，让系统更加稳定和美观。
