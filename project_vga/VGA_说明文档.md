# VGA 显示模块说明文档

**模块文件**：`project_vga/vga_display.v`  
**适用板卡**：EGO1（xc7a35tcsg324-1）  
**版本**：v1.0 · 2026-05-24

---

## 1. 功能概述

`vga_display` 模块通过 EGO1 板载 VGA 接口将售货机的实时状态输出到显示器。其显示内容与 8 位数码管完全同步——两者共享同一根 40 bit `disp_data` 总线，屏幕上的 8 个大字符与数码管显示的 8 位字符一一对应。

| 规格 | 参数 |
|------|------|
| 分辨率 | 640 × 480 |
| 刷新率 | 60 Hz |
| 像素时钟 | 25 MHz（100 MHz ÷ 4，片内分频）|
| 色深 | 12-bit（R/G/B 各 4 bit）|
| 同步极性 | HS/VS 均为**低有效**（负极性，标准 VGA）|

## 3. 字体

模块内置 **8×8 像素点阵字体**，在屏幕上以 **8× 比例** 放大显示（每个字体像素 = 8×8 屏幕像素），最终每字符占 64×64 px。

字符 ID 编码与 `seg7_mux.v` / `sales_v3.v` 完全一致：

| ID | 字符 | ID | 字符 | ID | 字符 |
|----|------|----|------|----|------|
| 0–9 | 数字 0–9 | 10 | A | 11 | b |
| 12 | C | 13 | d | 14 | E |
| 15 | F | 16 | H | 17 | L |
| 18 | n | 19 | o | 20 | S |
| 21 | t | 22 | r | 23 | 空格 |
| 24 | - | — | — | — | — |

字体数据存储在片内 LUTRAM（`rom_style = "distributed"`），确保无时钟延迟的组合读取。

## 4. 接口信号

```verilog
module vga_display (
    input  wire        clk,        // 100 MHz 系统时钟
    input  wire        rst_n,      // 低有效复位
    input  wire [39:0] disp_data,  // 8×5-bit 字符 ID，格式 {id7,id6,...,id0}
    input  wire [1:0]  sys_state,  // 2'd0=主菜单 2'd1=销售 2'd2=管理
    output wire [3:0]  vga_r,      // 红色通道
    output wire [3:0]  vga_g,      // 绿色通道
    output wire [3:0]  vga_b,      // 蓝色通道
    output wire        vga_hs,     // 行同步（低有效）
    output wire        vga_vs      // 场同步（低有效）
);
```

`disp_data` 的打包格式与 `seg7_mux` 输入一致：

```
disp_data[39:35] = id7（屏幕最左侧字符）
disp_data[34:30] = id6
…
disp_data[4:0]   = id0（屏幕最右侧字符）
```

## 5. 硬件连接

### 5.1 引脚分配（EGO1）

| 信号 | FPGA 引脚 | 电平标准 |
|------|----------|---------|
| `vga_r[0]` | F5 | LVCMOS33 |
| `vga_r[1]` | C6 | LVCMOS33 |
| `vga_r[2]` | C5 | LVCMOS33 |
| `vga_r[3]` | B7 | LVCMOS33 |
| `vga_g[0]` | B6 | LVCMOS33 |
| `vga_g[1]` | A6 | LVCMOS33 |
| `vga_g[2]` | A5 | LVCMOS33 |
| `vga_g[3]` | D8 | LVCMOS33 |
| `vga_b[0]` | C7 | LVCMOS33 |
| `vga_b[1]` | E6 | LVCMOS33 |
| `vga_b[2]` | E5 | LVCMOS33 |
| `vga_b[3]` | E7 | LVCMOS33 |
| `vga_hs`   | D7 | LVCMOS33 |
| `vga_vs`   | C4 | LVCMOS33 |

### 5.2 物理连接

将标准 VGA 线缆插入 EGO1 板上的 VGA D-Sub（15 针）接口，另一端连接支持 **640×480@60Hz** 的显示器即可。EGO1 板上已内置 DAC 电阻网络（R-2R 梯形网络），无需外部电路。

## 6. 时序说明

```
100 MHz 系统时钟
      ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐
      │ │ │ │ │ │ │ │ │ │
──────┘ └─┘ └─┘ └─┘ └─┘ └─

pclk_en（÷4 脉冲，25 MHz 等效像素时钟）
      ┌─┐               ┌─┐
      │ │               │ │
──────┘ └───────────────┘ └─

hc/vc 计数器在每个 pclk_en 脉冲递增。
RGB、HS、VS 输出在 100 MHz 时钟沿寄存，
比 hc/vc 滞后 1 个系统时钟（10 ns），
远小于像素周期（40 ns），对显示无影响。
```

VGA 640×480@60Hz 标准时序：

| 参数 | 数值 |
|------|------|
| 像素时钟 | 25 MHz（25.175 MHz 标准，误差 < 0.7 %）|
| 行总像素 | 800（640 有效 + 16 FP + 96 同步 + 48 BP）|
| 帧总行数 | 525（480 有效 + 10 FP + 2 同步 + 33 BP）|
| 实际刷新率 | 25 000 000 ÷ (800×525) ≈ 59.5 Hz |

## 7. 在 Vivado 中集成

### 7.1 添加源文件

在 Vivado Project Manager → **Add Sources** 中将以下文件加入项目：

```
project_vga/vga_display.v
```

### 7.2 确认约束文件已更新

打开 `project_top/vending.xdc`，确认 VGA 部分**已取消注释**（v1.0 起已默认启用）：

```tcl
set_property PACKAGE_PIN F5 [get_ports {vga_r[0]}]
...
set_property PACKAGE_PIN C4 [get_ports vga_vs]
```

### 7.3 顶层模块已自动更新

`drink_vending_top.v` 已包含 VGA 端口声明和 `u_vga` 实例，无需手动修改。

### 7.4 综合与实现

正常运行 **Synthesis → Implementation → Generate Bitstream**。  
VGA 模块资源估算（Artix-7 xc7a35t）：

| 资源 | 预估用量 | 板卡总量 |
|------|---------|---------|
| LUT（字体 ROM + 逻辑）| ~60 | 20,800 |
| FF（计数器 + 输出寄存器）| ~40 | 41,600 |
| BRAM | 0（使用 LUTRAM）| 50 |

## 8. 常见问题

**Q: 显示器无信号或不同步**  
A: 确认显示器支持 640×480@60Hz。部分显示器最低分辨率为 800×600；此时需修改 `vga_display.v` 中的时序常量为对应标准值。

**Q: 图像偏移或字符位置不对**  
A: EGO1 VGA 接口为无极性 D-Sub，确认线缆牢固连接。若偏移 1 列，检查输出寄存器是否被工具优化掉（可在综合选项中禁用 IOB packing）。

**Q: 颜色只有黑白**  
A: 检查 vga_r/vga_g/vga_b 的引脚分配是否正确，以及 IOSTANDARD 是否均设置为 LVCMOS33。

**Q: 如何修改背景颜色或字符颜色**  
A: 编辑 `vga_display.v` 中 `bg_r/g/b` 和 `bar_r/g/b` 的 case 语句，4-bit 值对应 0–F（16 级亮度）。

**Q: 如何在 VGA 上显示更多信息**  
A: 可在内容区（y=62..417，x=0..639）增加额外的字符行，需要在模块中添加第二套字符区域逻辑，并从顶层传入额外数据信号。
