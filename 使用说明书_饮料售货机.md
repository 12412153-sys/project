## 1. 系统总览

本系统支持 **4 种饮料**，具备 **销售** 和 **管理** 两种工作模式，通过板载 8 位数码管、16 个 LED、5 个按键、8 个拨码开关及 PS/2 键盘进行人机交互。

### 1.1 三大状态

```
上电/复位
    │
    ▼
MAIN_MENU（主菜单）
    ├── SW[0]=0，按 S2 ──▶ SALE_MODE（销售模式）
    │                           │ 购买完成或取消
    │                           ▼
    │                       MAIN_MENU
    │
    └── SW[0]=1，按 S2 ──▶ ADMIN_MODE（管理模式）
                                │ 退出或报警后确认
                                ▼
                            MAIN_MENU
```

### 1.2 系统默认初始数据（复位后恢复）

| 编号 | 饮料 | 数码管缩写 | 单价（¥） | 初始库存 | 初始状态 |
|:----:|:----:|:---------:|:--------:|:-------:|:-------:|
| 0 | COLA | `C0dA` | 4 | 5 | 在售 |
| 1 | SODA | `50dA` | 5 | 6 | 在售 |
| 2 | TEA  | `7EA ` | 3 | 8 | 在售 |
| 3 | H2O  | `b20 ` | 2 | 9 | 在售 |

- **累计营收**：¥0  
- **密码**：`42`（先按键盘 `4`，再按 `2`）

## 2. 硬件接口速查

### 2.1 按键（已消抖，上升沿有效）

| 板载标识 | 信号 | 引脚 | 销售模式功能 | 管理模式功能 | 主菜单功能 |
|:-------:|:----:|:----:|:-----------:|:-----------:|:---------:|
| **S2** | BTN[2] | R15 | 确认（选饮料 / 确认支付 / 确认取货） | 确认（密码 / 保存修改 / 消警报） | 进入所选模式 |
| **S0** | BTN[0] | R11 | 追加支付金额 | 下一属性（向右翻页） | — |
| **S1** | BTN[1] | R17 | 下一种饮料 | 饮料编号 −1 | — |
| **S3** | BTN[3] | V1  | 取消订单 | 上一属性（向左翻页）/ 退出管理 | — |
| **S4** | BTN[4] | U4  | 上一种饮料 | 饮料编号 +1 | — |
| **S6** | CPU_RESETN | P15 | — | — | 全局复位（低电平有效） |

> S6 按下（主动低电平）即全局复位，所有数据恢复默认值。

### 2.2 拨码开关

| 开关 | 引脚 | 作用 |
|:----:|:----:|:-----|
| SW[0] | R1 | **主菜单模式选择**：下拨=0（销售），上拨=1（管理） |
| SW[1] | N4 | **管理模式修改使能**：上拨=1 进入修改子状态 |
| SW[7:0] | R1…P5 | **销售模式支付金额输入**（8位二进制，范围 0–255） |

### 2.3 PS/2 键盘

仅在管理模式中使用，用于输入密码和修改数值。  
- 按数字键 `0`–`9` 即可输入；系统每次接收最后两位按键（先按高位，后按低位）。  
- 例如输入密码 `42`：先按 `4`，再按 `2`，再按 S2 确认。

### 2.4 数码管显示编码约定

| 显示字符 | 含义 |
|:--------:|:-----|
| `0`–`9`  | 数字 0–9 |
| `C` / `c` | 字母 C（COLA 首字母） |
| `d`      | 字母 d |
| `A` / `a` | 字母 A |
| `5`      | SODA 中的 S（七段近似） |
| `7`      | TEA 中的 t（七段近似） |
| `E`      | 字母 E |
| `b`      | 字母 b（H2O 中的 H 近似） |
| `_`（灭） | 空白占位 |
| `-`      | 横线 |
| `r`      | 字母 r |
| `S`      | 字母 S |

## 3. 上电与主菜单

**上电或按下 S6（CPU_RESETN）** 后，系统进入主菜单。

- **数码管显示**：`r1-0S-1A`  
  含义：拨 SW[0]=0 选销售模式（S），拨 SW[0]=1 选管理模式（A）

- **LED**：  
  - SW[0]=0（销售）：右侧 8 个 LED 点亮（`0x00FF`）  
  - SW[0]=1（管理）：左侧 8 个 LED 点亮（`0xFF00`）

- **操作**：调好 SW[0] 后按 **S2** 进入对应模式

## 4. 销售模式（SALE_MODE）

### 4.1 完整购买流程

```
进入销售模式
    │
    ▼
【SELECT 选择饮料】
  S4/S1 切换饮料，数码管显示饮料名称与价格
    │ 按 S2 确认选择
    ▼
【CHECK 检查库存】
  系统自动检查：库存 > 0 且在售
    ├── 通过 ──▶ 【PAY 支付】
    └── 不通过 ──▶ 返回 SELECT（显示提示）
    │
    ▼
【PAY 支付】
  拨动 SW[7:0] 设置金额，按 S0 追加投币，按 S2 确认支付
    ├── 金额 ≥ 单价 ──▶ 【DISPENSE 出货】
    └── 金额不足 ──▶ 留在 PAY（可继续追加）
    │
    ▼
【DISPENSE 出货中】
  系统自动出货，等待动作完成
    │
    ▼
【WAIT_TAKE 等待取货】
  数码管/LED 提示取货
    │ 按 S2 确认已取走
    ▼
【COMPLETE 完成】
  ★ 此刻写入寄存器：库存 −1，营收 + 该饮料价格
  自动返回主菜单
    │
    ▼（若已投金额 > 单价）
【REFUND 退款】
  数码管显示找零金额，确认后返回主菜单
```

**任意阶段按 S3** = 取消订单，退款已投金额，返回主菜单。

### 4.2 选择饮料界面说明

| 操作 | 效果 |
|:----:|:-----|
| S4 | 切换到上一种饮料 |
| S1 | 切换到下一种饮料 |
| S2 | 确认当前饮料，进入检查 |
| S3 | 取消，返回主菜单 |

- 只有**在售（enabled=1）且库存 > 0** 的饮料才能购买
- 数码管显示：饮料缩写名 + 当前单价（来自寄存器实时读取）

### 4.3 支付界面说明

| 操作 | 效果 |
|:----:|:-----|
| 调整 SW[7:0] | 设置本次追加投币金额（0–255） |
| S0 | 将 SW[7:0] 当前值累加到已投金额 |
| S2 | 确认支付（需已投金额 ≥ 单价） |
| S3 | 取消，退还已投金额，返回主菜单 |

## 5. 管理模式（ADMIN_MODE）

### 5.1 进入管理模式与密码认证

1. 在主菜单将 **SW[0] 上拨（=1）**，按 **S2** 进入管理模式
2. 数码管显示：`----C0dE`（提示输入密码）
3. 在 PS/2 键盘上依次按下密码的**两位数字**（先高位，后低位）
   - 默认密码：先按 `4`，再按 `2`
4. 按 **S2** 确认
   - **密码正确** → 进入查看界面（S_VIEW）
   - **密码错误** → 错误计数 +1，数码管重新显示 `----C0dE`，可重新输入
   - **连续输错 3 次** → 进入报警状态

### 5.2 报警状态（S_ALARM）

连续输错密码 **3 次**后：

- 数码管显示：`AAAAAAAA`  
- 蜂鸣器报警响  
- LED 显示交替闪烁图案（`0xAAAA`）

**解除方式**：按 **S2** → 蜂鸣器停止，自动退出管理模式返回主菜单，错误计数清零。

### 5.3 查看界面（S_VIEW）

密码正确后进入查看界面，可逐属性浏览所有饮料参数和营收。

#### 浏览顺序（按 S0 向右翻页）

```
COLA-库存 → COLA-单价 → COLA-状态
    → SODA-库存 → SODA-单价 → SODA-状态
    → TEA-库存  → TEA-单价  → TEA-状态
    → H2O-库存  → H2O-单价  → H2O-状态
    → 累计营收
```

按 **S3 向左翻页**（逆序），在第一项（COLA 库存）再按 S3 → **退出管理模式**，返回主菜单。

#### 按键说明（查看界面）

| 按键 | 功能 |
|:----:|:-----|
| S0 | 下一个属性（向右翻页） |
| S3 | 上一个属性（向左翻页）；在首项时退出管理模式 |
| S4 | 直接跳到下一种饮料（编号 +1，最大 3） |
| S1 | 直接跳到上一种饮料（编号 −1，最小 0） |
| SW[1] 上拨 | 进入修改模式（在"累计营收"页面时无效） |

#### 数码管显示格式（查看界面）

| 属性 | 4字符饮料（COLA/SODA）显示 | 3字符饮料（TEA/H2O）显示 |
|:----:|:------------------------:|:----------------------:|
| 库存 | `C0dA__05`（库存=5） | `7EA___08`（库存=8） |
| 单价 | `C0dA__04`（单价=¥4） | `7EA___03`（单价=¥3） |
| 在售状态 | `C0dA___1`（在售）/ `C0dA___0`（停售） | `7EA____1` |
| 累计营收 | 左侧4位显示营收十六进制值，右侧4位空白 | 同左 |

> **状态位说明**：`1` = 在售，`0` = 停售（即禁用）。

### 5.4 修改界面（S_MODIFY）

在查看界面将 **SW[1] 上拨（=1）** 即进入修改界面（仅限非营收页面）。

- 数码管左侧显示饮料缩写名，右侧两位显示**当前键盘缓冲区数值**（两位十进制）
- 在 PS/2 键盘上依次按下新数值的数字（例如要设置为 7，按 `0` 再按 `7`，显示 `07`）
- 按 **S2** 保存

| 当前属性 | 修改内容 | 说明 |
|:--------:|:-------:|:-----|
| 库存 | 设置库存到指定值 | 输入 0–9（输入值 >9 自动截止为 9） |
| 单价 | 设置单价到指定值 | 输入 0–9 |
| 在售状态 | 切换在售/停售 | 直接按 S2，无需键盘输入（每次反转） |

**退出修改模式**：将 SW[1] 下拨（=0）→ 自动返回查看界面，未保存的修改丢弃。

### 5.5 退出管理模式

以下任一操作退出管理模式，返回主菜单：

- 在查看界面的**第一项（COLA 库存）**按 **S3**
- 报警状态下按 **S2** 消警

## 6. LED 状态说明

| 系统状态 | LED 显示（16位，高位=左） | 含义 |
|:--------:|:------------------------:|:-----|
| 主菜单，SW[0]=0 | `0x00FF`（右8亮） | 提示当前选择：销售模式 |
| 主菜单，SW[0]=1 | `0xFF00`（左8亮） | 提示当前选择：管理模式 |
| 销售模式 | 由销售模式驱动 | 显示支付/找零/出货等状态 |
| 管理模式，正常 | 全灭（`0x0000`） | 无报警 |
| 管理模式，报警中 | `0xAAAA`（交替闪烁） | 密码连续错误 3 次 |

## 7. 典型操作示例

### 示例一：购买一瓶 COLA（¥4）

1. 上电，数码管显示 `r1-0S-1A`，SW[0] 保持下拨（=0）
2. 按 **S2** → 进入销售模式
3. 数码管显示当前饮料名称和价格；按 **S1** / **S4** 切换到 COLA
4. 按 **S2** 确认选择 → 系统检查库存（COLA 初始库存 5，通过）
5. 进入支付界面，调 SW[7:0] = `0000 0100`（=4），按 **S0** 追加 ¥4
6. 按 **S2** 确认支付（¥4 ≥ 单价 ¥4）→ 出货
7. 数码管提示取货，按 **S2** 确认取走
8. ★ 寄存器更新：COLA 库存变为 4，营收 +4
9. 自动返回主菜单

### 示例二：查看并修改 SODA 单价（改为 ¥6）

1. 主菜单 SW[0] 上拨（=1），按 **S2** → 进入管理模式
2. 数码管显示 `----C0dE`，键盘按 `4` 再按 `2`，按 **S2** 确认密码
3. 进入查看界面，默认显示 COLA 库存（`C0dA__05`）
4. 按 **S0**（下一属性）→ 显示 COLA 单价（`C0dA__04`）
5. 按 **S0** → 显示 COLA 状态（`C0dA___1`）
6. 按 **S0** → 显示 SODA 库存（`50dA__06`）
7. 按 **S0** → 显示 SODA 单价（`50dA__05`）
8. 将 **SW[1] 上拨（=1）** → 进入修改界面（数码管右侧变为 `00`）
9. 键盘按 `0` 再按 `6`（数码管右侧显示 `06`）
10. 按 **S2** 保存 → SODA 单价更新为 ¥6
11. 将 SW[1] 下拨（=0）→ 返回查看界面，显示 `50dA__06`（已更新）
12. 按 **S3** 逐步翻回到第一项，在 COLA 库存页面按 **S3** → 退出管理模式

### 示例三：停售 TEA

1. 进入管理模式（同示例二步骤 1–2）
2. 在查看界面按 **S4** 将饮料 ID 切换到 2（TEA）
3. 按 **S0** 两次翻到 TEA 在售状态页（`7EA____1`）
4. 将 **SW[1] 上拨（=1）** → 进入修改界面
5. 直接按 **S2**（无需键盘输入）→ 状态反转为停售
6. 修改界面返回后显示 `7EA____0`（已停售）
7. 退出管理模式（SW[1] 下拨 → S3 翻回首页 → S3 退出）
8. 进入销售模式后，TEA 将无法被选购

### 示例四：查看累计营收

1. 进入管理模式，输入密码
2. 在查看界面连续按 **S0** 翻过所有 4 种饮料的 3 个属性（共 12 次）
3. 进入营收页面：左侧显示当前营收的十六进制值，右侧 4 位空白
   - 例如营收 = ¥20（0x0014）时显示 `0014____`
4. 按 **S3** 返回上一页，继续翻回首页后按 **S3** 退出

## 8. 注意事项

1. **复位清零**：按下 S6 会将所有参数（价格、库存、营收、密码）恢复为出厂默认值。

2. **密码修改**：当前版本密码固定为 `42`，**不支持通过界面修改密码**（密码存储在寄存器文件，出厂值 `0x42`，只能通过复位恢复）。

3. **密码输入规则**：PS/2 键盘输入时，系统保存**最后两次**按下的数字作为密码（先按的是高位）。例如不小心按了 `1`、`3`、`4`、`2`，系统读取的是最后两次 → `42`，仍然正确。

4. **库存上限**：单种饮料库存最大值为 **9**，设置超过 9 时自动截为 9。

5. **支付金额**：SW[7:0] 范围 0–255，按 S0 追加后累计，不可减少（需取消订单重来）。

6. **购买限制**：停售（enabled=0）的饮料即使有库存也无法购买；库存为 0 的饮料也无法购买。

7. **同步机制**：销售模式完成购买后，库存和营收**立即写入中央寄存器**，切换回管理模式查看的数值即为最新值，两个模式共享同一份数据存储。

8. **报警解除**：3 次密码错误触发报警后，只能通过按 S2 解除，解除后**直接返回主菜单**（不进入查看界面），错误计数清零，可重新进入管理模式。
