## 1. 系统总览

本系统支持 **4 种饮料**，具备购物车式**销售模式**（一次可购多种/多瓶）和**管理模式**，通过板载 8 位数码管、16 个 LED、5 个按键、2 个拨码开关及 PS/2 键盘进行人机交互。

### 1.1 三大状态

```
上电/重新配置（S5·PROG_B）
        │
        ▼
  MAIN_MENU（主菜单）
        ├── SW[0]=0，按 S2 ──▶ SALE_MODE（销售模式）
        │                           │ 按 S6 软退出（数据保留）
        │                           ▼
        │                       MAIN_MENU
        │
        └── SW[0]=1，按 S2 ──▶ ADMIN_MODE（管理模式）
                                    │ 按 S6 软退出（数据保留）
                                    ▼
                                MAIN_MENU
```

> **两种模式均通过 S6 键软退出**（数据保留，不清除价格/库存/营收/密码）。  
> **S5（PROG_B）** 按下时 FPGA 重新加载比特流，等效于完全上电复位，所有数据恢复默认值。

### 1.2 系统默认初始数据（重新配置后恢复）

| 编号 | 饮料 | 数码管缩写 | 单价（¥） | 初始库存 | 初始状态 |
|:----:|:----:|:---------:|:--------:|:-------:|:-------:|
| 0 | COLA | `C0dA` | 4 | 5 | 在售 |
| 1 | SODA | `50dA` | 5 | 6 | 在售 |
| 2 | TEA  | `7EA ` | 3 | 8 | 在售 |
| 3 | H2O  | `b20 ` | 2 | 9 | 在售 |

- **累计营收**：¥0  
- **密码**：`42`（先按键盘 `4`，再按 `2`）

## 2. 硬件接口速查

### 2.1 按键（已消抖，上升沿有效）

| 板载 | 引脚 | 销售模式功能 | 管理模式功能 | 主菜单功能 |
|:---:|:----:|:-----------:|:-----------:|:---------:|
| **S2** | R15 | 确认选饮料 / 确认数量 / 确认付款 / 确认取货 | 确认密码 / 保存修改 / 解除报警 | 进入所选模式 |
| **S6** | P15（低有效） | **软退出→主菜单（数据保留）** | **软退出→主菜单（数据保留）** | — |
| **S1** | R17 | 下一种饮料 | 饮料编号 −1 | — |
| **S4** | U4  | 上一种饮料 | 饮料编号 +1 | — |
| **S3** | V1  | **结算 / 进入付款** | 属性向左翻页 | — |
| **S0** | R11 | 清空购物车（选择页） / 取消付款→退款 | 属性向右翻页 | — |
| **S5** | PROG_B（专用配置引脚） | 全局复位（重新加载比特流，**所有数据归零**） | ← | ← |

### 2.2 拨码开关

| 开关 | 引脚 | 作用 |
|:----:|:----:|:-----|
| SW[0] | R1 | **主菜单模式选择**：下拨=0（销售），上拨=1（管理） |
| SW[1] | N4 | **管理模式修改使能**：上拨=1 进入修改子状态 |
| SW[2..7] | — | 当前版本未使用 |

### 2.3 PS/2 键盘

两种模式均使用 PS/2 键盘输入数字（`0`–`9`）：

| 模式 | 用途 |
|:----:|:-----|
| 销售模式 | 输入**购买数量**（ST_QTY）和**支付金额**（ST_PAY） |
| 管理模式 | 输入**密码**（S_AUTH）和**修改值**（S_MODIFY） |

系统每次接收最后若干按键，高位先按。例如输入金额 `15`：先按 `1` 再按 `5`，显示 `15`。输入超过最大值（数量 99，金额 999）时自动截为最大值。

### 2.4 数码管字符编码速查

| 显示 | 含义 |
|:----:|:-----|
| `0`–`9` | 数字 |
| `C` `0` `d` `A` | COLA 缩写 |
| `5` `0` `d` `A` | SODA 缩写（`5` 近似 S） |
| `7` `E` `A` `_` | TEA 缩写（`7` 近似 t，`_`=空） |
| `b` `2` `0` `_` | H2O 缩写（`b` 近似 H） |
| `_`（全灭） | 空白占位 |
| `-` | 横线 |

## 3. 上电与主菜单

**上电或按下 S5（PROG_B）** 后，系统进入主菜单：

- **数码管显示**：`r1-0S-1A`  
  读法：SW[0]=0 → 右半段 `0S-1A` 表示选销售，SW[0]=1 → 左半段 `r1-0` 表示选管理

- **LED**：  
  - SW[0]=0（销售）：右侧 8 个 LED 亮（`0x00FF`）  
  - SW[0]=1（管理）：左侧 8 个 LED 亮（`0xFF00`）

- **操作**：调好 SW[0]，按 **S2** 进入

## 4. 销售模式（SALE_MODE）

### 4.1 状态流程总览

```
进入销售模式（sales_en 有效）
        │
        ▼
【ST_SELECT 选择饮料】
  S1/S4 翻页，S2 选饮料，S3 结算（购物车非空），S0 取消回主菜单
        │ S2 选中某饮料
        ▼
【ST_CHECK 检查】（自动）
  检查在售、库存 > 0、价格合法
        │ 通过         │ 不通过
        ▼              ▼
【ST_QTY 输入数量】  【ST_ERROR 错误提示】─▶ 自动回 ST_SELECT
  键盘输入数量（0–99），S2 确认加入购物车
        │ 数量超出剩余库存         │ 订单总价超 999
        ▼                        ▼
【ST_STOCK 库存不足提示】    【ST_FULL 超额提示】
  自动/S2/S0 → 回 ST_SELECT    自动/S2/S0 → 回 ST_SELECT
        │ 加入成功
        ▼
【ST_ADDED 已加入提示】
  显示当前购物车总价
  S3 → 进入付款；S2/S0/超时 → 回 ST_SELECT 继续选

        ─────── 按 S3（结算）任意时刻从 ST_SELECT 进入 ───────
        ▼
【ST_PAY 付款】
  键盘输入金额（0–999），S2 确认
        │ 金额不足           │ 刚好         │ 超额
        ▼                    ▼              ▼
  显示 Add xxx        【ST_WAIT 取货】  【ST_CHANGE 找零】
  继续输入                  │            S2/超时 → ST_WAIT
                            │ S2 确认取走
                            ▼
                    【ST_COMMIT 写库存/收入】
                      每瓶依次扣库存并计入营收
                            │
                            ▼
                    【ST_DONE 完成】─▶ 自动回 ST_SELECT

  S0（取消）在付款/等待阶段 → 【ST_REFUND 退款】─▶ 自动回 ST_SELECT
  ST_WAIT 超时 → 退款订单总额，不扣库存
```

### 4.2 ST_SELECT 选择饮料

| 操作 | 效果 |
|:----:|:-----|
| S1 | 下一种饮料（循环） |
| S4 | 上一种饮料（循环） |
| S2 | 确认选中当前饮料，进入检查 |
| S3 | 结算（仅购物车非空时有效）→ 进入付款 |
| S0 | 清空购物车，保持在选择页 |
| S6 | 软退出，返回主菜单（购物车/订单数据清零，寄存器数据保留） |

**数码管显示**：`饮料缩写 + 单价 + 空格 + 剩余可购库存（2位）`

```
COLA4 05   ← COLA 单价4，剩余库存5
50dA5 06   ← SODA 单价5，剩余库存6
7EA 3 08   ← TEA  单价3，剩余库存8
b20 2 09   ← H2O  单价2，剩余库存9
```

> 剩余库存 = 实际库存 − 购物车中该饮料数量，即已选入购物车的数量已从显示库存中扣除。

### 4.3 ST_QTY 输入数量

| 操作 | 效果 |
|:----:|:-----|
| 键盘 `0`–`9` | 输入数量（最多2位，上限99） |
| S2 | 确认，尝试加入购物车 |
| S0 | 取消当前数量输入，返回 ST_SELECT（购物车保留） |

**数码管显示**：`Cnt当前输入数量 P单价`  
例：买3瓶COLA（单价4）→ `Cnt03 P4`

加入成功后显示：`tot 总价`  
例：总价12 → `tot 012`（此时可按 S3 直接去付款，或 S2/S0 继续选其他饮料）

### 4.4 ST_PAY 付款

| 操作 | 效果 |
|:----:|:-----|
| 键盘 `0`–`9` | 累积输入金额（最多3位，上限999） |
| S2 | 确认支付 |
| S0 | 取消订单，退还已输入金额，进入 ST_REFUND |

**数码管显示**：

| 情况 | 显示 | 含义 |
|:----:|:----:|:-----|
| 正常输入 | `XXXXtYYY` | 已付 XXXX，总价 YYY（t 为分隔符） |
| 金额不足（按S2后） | `Add  YYY` | 还需补 YYY |
| 找零（按S2后） | `rEt  YYY` | 找零 YYY，稍后自动进入等待取货 |

金额不足时可继续键入更多数字（累加），再次按 S2 重新校验。  
**支付超时（40秒无操作）**：自动退款已投金额，进入 ST_REFUND。

### 4.5 ST_WAIT 等待取货

- **数码管显示**：`PUSH    `（提示按 S2 取走饮料）
- **LED**：流水灯，越接近超时（5秒）流速越快

| 操作 | 效果 |
|:----:|:-----|
| S2 | 确认取走 → 进入 ST_COMMIT（扣库存并计入营收） |
| S0 | 取消，退款订单总价，不扣库存 |
| 超时（5秒） | 自动退款，不扣库存 |

### 4.6 ST_COMMIT 写入寄存器

- **数码管显示**：` OUt    `
- **LED**：全亮（`0xFFFF`）
- 按购物车内容逐瓶触发 `sale_we`，寄存器文件每收一次脉冲扣1瓶库存并加一笔收入
- 所有项目处理完后自动进入 ST_DONE

### 4.7 结束页面

| 状态 | 数码管 | LED | 含义 |
|:----:|:------:|:---:|:-----|
| ST_DONE   | `dOnE    ` | `0xAAAA` | 交易成功 |
| ST_REFUND | `rEt  YYY` | `0x5555` | 退款 YYY 元 |
| ST_ERROR  | `Err     n` | `0xF00F` | 错误码 n（见下） |

**错误码**：

| 错误码 | 含义 |
|:------:|:-----|
| 1 | 饮料已停售（enabled=0） |
| 2 | 库存不足（stock=0 或购物车已选满） |
| 3 | 金额不足（预留，实际由 Add 提示） |
| 4 | 价格非法（price=0 或 >9） |
| 5 | 超时 |

## 5. 管理模式（ADMIN_MODE）

### 5.1 进入与密码认证

1. 主菜单 SW[0] 上拨（=1），按 **S2** 进入管理模式
2. **数码管显示**：`----C0dE`
3. PS/2 键盘依次按下两位密码（先高位）→ 按 **S2** 确认
   - 默认密码：`4` 然后 `2`
4. **结果**：
   - 正确 → 进入查看界面
   - 错误 → 错误计数 +1，重新显示 `----C0dE`，可重新输入
   - **连续错误 3 次** → 进入报警（S_ALARM）

### 5.2 报警（S_ALARM）

- **数码管**：`AAAAAAAA`  
- **蜂鸣器**：持续报警  
- **LED**：`0xAAAA`（交替闪烁）

**解除**：按 **S2** → 蜂鸣器停止，**自动返回密码输入界面**（错误计数清零，仍处于管理模式）。如需退出管理模式，按 **S6** 软退出。

### 5.3 查看界面（S_VIEW）

密码正确后进入，可逐属性浏览所有饮料参数。

#### 浏览顺序

```
← S3（向左）      S0 向右 →

[COLA 库存] → [COLA 单价] → [COLA 状态]
→ [SODA 库存] → [SODA 单价] → [SODA 状态]
→ [TEA  库存] → [TEA  单价] → [TEA  状态]
→ [H2O  库存] → [H2O  单价] → [H2O  状态]
→ [累计营收]
```

在 **COLA 库存**（第一项）按 **S3**：数码管**保持不变**（不退出，不导航）。

#### 按键说明

| 按键 | 功能 |
|:----:|:-----|
| S0 | 下一个属性（向右翻） |
| S3 | 上一个属性（向左翻）；在首项保持不变 |
| S4 | 直接跳到下一种饮料（编号 +1，最大 3） |
| S1 | 直接跳到上一种饮料（编号 −1，最小 0） |
| SW[1] 上拨 | 进入修改模式（营收页面无效） |

#### 数码管显示格式

| 属性 | 4字符饮料 COLA/SODA | 3字符饮料 TEA/H2O |
|:----:|:------------------:|:----------------:|
| 库存 | `C0dA__05`（库存5） | `7EA___08`（库存8） |
| 单价 | `C0dA__04`（单价¥4） | `7EA___03`（单价¥3） |
| 状态 | `C0dA___1`（在售）/ `C0dA___0`（停售） | `7EA____1` |
| 累计营收 | 左4位=营收十六进制，右4位空白 | 同左 |

> **状态位**：`1` = 在售，`0` = 停售。

### 5.4 修改界面（S_MODIFY）

在查看界面将 **SW[1] 上拨（=1）** 进入（营收页无效）。

- 数码管左侧：饮料缩写名
- 数码管右两位：键盘缓冲区当前数值（两位十进制）
- 按键盘数字键输入新值 → 按 **S2** 保存并返回查看

| 属性 | 修改内容 | 说明 |
|:----:|:-------:|:-----|
| 库存 | 设置库存为指定值 | 输入 0–9；超过9自动截为9 |
| 单价 | 设置单价为指定值 | 输入 0–9 |
| 在售状态 | 切换在售 / 停售 | **直接按 S2**（无需键盘），每次翻转一次 |

将 SW[1] 下拨（=0）退出修改，未保存的修改丢弃。

### 5.5 退出管理模式

| 方式 | 操作 | 数据是否保留 |
|:----:|:-----|:------------:|
| **S6 软退出** | 任意时刻按 **S6**（低有效）→ 立即返回主菜单 | ✅ 保留 |
| **全局复位** | 按 **S5（PROG_B）** → FPGA 重新加载比特流，所有数据恢复默认值 | ❌ 归零 |

> 报警解除（S2 in S_ALARM）返回密码输入界面，仍处于管理模式。如需退出请按 **S6**。

## 7. 典型操作示例

### 示例一：购买 2 瓶 COLA + 3 瓶 TEA（混合购物车）

**初始状态**：上电后主菜单，SW[0] 下拨（=0）

| 步骤 | 操作 | 数码管显示 | 说明 |
|:----:|:----:|:----------:|:-----|
| 1 | 按 **S2** | `COLA4 05` | 进入销售模式，默认显示COLA，库存5 |
| 2 | 按 **S2** | ` CHEC   ` | 选中COLA，自动检查（在售√ 库存>0√ 价格√） |
| 3 | （等待）→ `Cnt00 P4` | `Cnt00 P4` | 自动跳入数量输入，显示当前输入0，单价4 |
| 4 | 键盘按 `2` | `Cnt02 P4` | 输入数量2 |
| 5 | 按 **S2** | `tot 008` | 2瓶COLA加入购物车（2×4=8），显示总价8 |
| 6 | 按 **S2** 或 **S0**（返回选饮料） | `COLA4 03` | 回到选择页，COLA剩余可购库存变为 5−2=3 |
| 7 | 按 **S1** | `50dA5 06` | 切换到SODA |
| 8 | 按 **S1** | `7EA 3 08` | 切换到TEA |
| 9 | 按 **S2** | ` CHEC   ` | 选中TEA，检查通过 |
| 10 | 键盘按 `3` | `Cnt03 P3` | 输入数量3 |
| 11 | 按 **S2** | `tot 017` | 3瓶TEA加入购物车（3×3=9），总价8+9=17 |
| 12 | 按 **S3** 结算 | `0000t017` | 进入付款页，当前已付0，总价17 |
| 13 | 键盘按 `2` `0` | `0020t017` | 输入金额20 |
| 14 | 按 **S2** 确认 | `rEt  003` | 金额20 > 总价17，找零3，持续约3秒 |
| 15 | （自动）→ `PUSH    ` | `PUSH    ` | 进入等待取货，流水灯开始 |
| 16 | 按 **S2** 取走饮料 | ` OUt    ` | 开始写入：2×COLA扣库存/加营收，3×TEA同 |
| 17 | （自动）→ `dOnE    ` | `dOnE    ` | 交易完成，LED 0xAAAA 闪烁 |
| 18 | 按 **S6** 软退出 | `r1-0S-1A` | 返回主菜单，营收/库存已更新且保留 |

**寄存器更新结果**：COLA库存 5→3，TEA库存 8→5，营收 0→17。

### 示例二：购买 1 瓶 H2O（刚好付款）

| 步骤 | 操作 | 数码管显示 | 说明 |
|:----:|:----:|:----------:|:-----|
| 1 | 按 **S2** | `COLA4 05` | 进入销售 |
| 2 | 按 **S1** 三次 | `b20 2 09` | 切换到H2O（单价2，库存9） |
| 3 | 按 **S2** | ` CHEC   ` | 选中H2O |

## 8. 注意事项

1. **退出任意子模式**：按 **S6**（低有效）软退出，返回主菜单，价格/库存/营收/密码全部保留。按 **S5（PROG_B）** 全局复位，FPGA 重新加载比特流，所有数据归零（相当于完全重新上电）。两者均可在销售模式和管理模式任意时刻使用。

2. **购物车保留**：每次进入销售模式购物车自动清空；取消（S0）也会清空购物车。

3. **密码输入规则**：系统记录键盘最后两次按键作为密码。例如多按了几个键，只要最后两位是 `4`、`2` 即可通过。

4. **库存显示**：选择页显示的是**扣除购物车后的剩余库存**，已在购物车的数量不显示在可购库存中。

5. **同步机制**：管理模式修改价格/库存/状态后，回到销售模式读取的是最新值；购买完成（ST_COMMIT 结束）后，营收和库存立即更新，下次进入管理模式查看即为最新值。

6. **库存上限**：寄存器文件库存最大值 9（管理员设置时输入 >9 自动截为 9）。

7. **单价限制**：价格需设置为 1–9。设置为 0 或 >9 时购买时会提示 `Err    4`（价格非法）。

8. **总价上限**：单次订单总价不超过 999。超出时显示 `FULL 999` 并提示重新选择数量。

9. **营收显示**：管理模式营收页以十六进制显示（如 ¥20 显示为 `0014____`）。
